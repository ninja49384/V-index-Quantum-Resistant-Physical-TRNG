## Supported Versions
We actively secure the latest main branch and the most recent release.

## Reporting a Vulnerability
If you discover a security issue, **do not** open a public issue.
Instead, email **kisnorbert87@gmail.com** with:
- A detailed description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested remediation (if any)

We aim to acknowledge reports within **72 hours** and provide a remediation plan or timeline within **7 days**. After a fix is released, we may publish a security advisory.

## Scope
- Client-side code running in the browser
- Entropy capture and validation (V-index)
- Key generation pipeline and integration points

## Out of Scope (examples)
- Third-party packages not maintained by this repo
- User environments (host OS, hardware quirks) unless directly impacted by our code

## Responsible Disclosure
Please avoid public disclosure until we release a fix and advisory. We appreciate coordinated disclosure and will credit reporters unless anonymity is requested.
